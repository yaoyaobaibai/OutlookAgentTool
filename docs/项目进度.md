## 🔜 下一阶段：PR/PO 自动化（未开始）

### 整体目标
从邮件接收 → AI 提取 → Acubuy 录入 → SharePoint 台账更新，端到端自动化。

### 8 个规划模块（按依赖顺序）

| # | 模块 | 职责 | 技术栈 | 状态 |
|---|------|------|--------|------|
| 1 | **Mail Agent** | 监听邮箱 + 下载附件 | Outlook COM / IMAP | ❌ 待开发 |
| 2 | Extract Agent | OCR + AI 提取金额/账期 | PaddleOCR / Qwen-VL | ❌ 待开发 |
| 3 | Decision Agent | 金额阈值判断（7.5w） | 纯 Python | ❌ 待开发 |
| 4 | Acubuy Agent | ivalua 系统自动填表 | Playwright | ❌ 待开发 |
| 5 | Notify Agent | 发邮件给审批人 | SMTP | ❌ 待开发 |
| 6 | SharePoint Agent | 更新台账 + 上传 | Graph API | ❌ 待开发 |
| 7 | PDF Agent | 合并 PR 号码到 PDF | pypdf | ❌ 待开发 |
| 8 | LangGraph 编排 | 串联所有节点 | LangGraph | ❌ 待开发 |

### 已确认的 7 项决策（Mail Agent 阶段）

| 问题 | 决定 |
|------|------|
| 协议 | 复用 Outlook COM（IMAP+IDLE 留待后续）|
| 账号 | 个人 Outlook |
| 过滤 | Python 独立规则引擎 + YAML 配置 |
| 轮询 | 2 分钟 |
| 附件处理 | **仅下载，不分类**（分类交给 Extract Agent）|
| 上下游交接 | inbox 目录轮询（避免函数耦合）|
| 数据出境 | MVP 阶段保留原文；日志仅含元数据 |

### 业务红线
- **绝对不允许**采购数据出境（金额/供应商/成本中心不许传 OpenAI 等公有云）
- 必须全程在企业内网处理
- AI 模型只能用本地部署（Qwen-VL via Ollama / PaddleOCR）

## 🛠️ 开发环境

### 必需
- Python 3.10+
- Windows 10/11（必须，Outlook COM 限制）
- MSYS2 + pango（构建 weasyprint 依赖）
- Outlook 客户端（已登录的 Outlook 账号）
- PyInstaller, pip

### 构建路径
```powershell
$env:PATH = "C:\msys64\mingw64\bin;" + $env:PATH
pyinstaller --onefile --windowed ...
```

## ⚠️ 重要约束（违反会出 bug）

1. **subprocess 必须隐藏窗口**：`creationflags=0x08000000`
2. **CSS 先清后注**：避免 xhtml2pdf 卡死
3. **`attachment_handler.py` 和 `msg_to_pdf.py` 同步修改**：两份相同代码
4. **所有日志纯英文 ASCII**：`--windowed` EXE 没有 stderr，中文变 `??`
5. **`@page` CSS 用 mm 不用 pt**：xhtml2pdf 对 pt 支持不全
6. **Win32 COM 后必须杀僵尸进程**：`taskkill /f /im WINWORD.EXE`

详见 `.sisyphus/skills/SKILL.md`
