# FormFiller v1.2 更新说明

## 🎯 本次更新修复的问题

### ✅ 1. Proposal # 和 GET CRM INFO 按钮处理
**问题**：填写 Proposal # 后没有自动点击 GET CRM INFO 按钮

**解决方案**：
- 程序现在会自动检测 "Proposal #" 字段
- 填写后自动点击 "GET CRM INFO" 按钮
- 等待 CRM 数据加载完成（等待 Loading 动画消失）
- 超时时间：15 秒

**日志输出**：
```

  已填写 Proposal #: P12345
  正在点击 GET CRM INFO 按钮...
  等待 CRM 数据加载...
  ✓ CRM 数据加载完成
```

### ✅ 2. Date of Award 日期选择器
**问题**：日期字段不是直接输入，而是通过蓝色按钮选择

**解决方案**：
- 检测 "Date of Award" 字段
- 自动点击右边的蓝色按钮打开日期选择器
- 在弹窗中填写日期（格式：MMDDYYYY）
- 如果找不到蓝色按钮，直接填写日期输入框

**日志输出**：
```
>>> 步骤 2: 选择 Date of Award
  点击日期选择按钮...
  已填写日期：01/15/2024
```

**注意**：Excel 中的日期格式必须是 `MM/DD/YYYY`

### ✅ 3. Priming Project Manager 弹窗搜索
**问题**：Project Manager 需要通过弹窗搜索选择

**解决方案**：
- 检测包含 "Project Manager" 或 "Priming Project Manager" 的字段
- 自动点击放大镜按钮打开搜索弹窗
- 在弹窗中输入 LoginID
- 点击 Search 按钮
- 选择第一个搜索结果
- 点击 Select 按钮确认

**日志输出**：

```
  已输入 LoginID: lin.zu
  已点击 Search 按钮
  已选择第一个搜索结果
  已点击 Select 按钮
```

### ✅ 4. 下拉框处理（Currency Code 等）
**问题**：Selling Price Currency Code 是下拉框，不能使用 fill() 方法

**错误信息**：
```
Locator.fill: Error: Element is not an <input>, <textarea> or [contenteditable] element
```

**解决方案**：
- 检测元素类型（`<select>`, `<input>`, `<textarea>`）
- 对 `<select>` 元素使用 `select_option()` 方法
- 对 `<input>` 和 `<textarea>` 使用 `fill()` 方法

**日志输出**：
```
>>> 步骤 4: 填充其他表单字段
  选择下拉框 'Selling Price Currency Code': USD
  已填充字段 'Cust Ref. No': CR-2024-001
  已填充字段 'Proposal/Contract Value': 500000
```

## 🔄 完整的自动化流程

现在程序会按以下顺序执行：

### 1️⃣ 登录阶段
- 打开浏览器
- 访问登录页面
- 自动填写用户名和密码
- 点击登录按钮
- 等待登录完成

### 2️⃣ 导航阶段
- 自动导航到 Create Proposal Group 页面
- 等待页面加载完成

### 3️⃣ Proposal # 处理
- 填写 Proposal #
- 点击 GET CRM INFO 按钮
- 等待 CRM 数据加载完成

### 4️⃣ Date of Award 处理
- 点击蓝色日期选择按钮
- 在弹窗中填写日期
- 关闭弹窗

### 5️⃣ Project Manager 处理
- 点击放大镜按钮打开弹窗
- 输入 LoginID
- 点击 Search
- 选择第一个结果
- 点击 Select 确认

### 6️⃣ 其他字段填充
- 检测元素类型（下拉框/输入框）
- 使用正确的方法填充
- 支持的文件类型：
  - `<select>` → `select_option()`
  - `<input>` → `fill()`
  - `<textarea>` → `fill()`

### 7️⃣ 附件上传
- 滚动到附件上传区域
- 对每个附件：
  - 选择 Category
  - 上传文件
  - 填写 Description

### 8️⃣ 完成
- 显示完成消息
- 等待用户检查
- 用户手动点击 Create 按钮

## 🐛 已知问题

### 问题 1：日期选择器弹窗
如果日期选择器弹窗无法正常打开，可能是因为：
- 浏览器阻止了弹窗
- 页面 JavaScript 加载未完成

**解决方法**：
- 允许浏览器弹出窗口
- 增加等待时间

### 问题 2：附件上传选择器
不同页面的附件上传区域选择器可能不同。

**解决方法**：
- 程序已尝试多种选择器
- 如果都失败，需要手动查看页面 HTML 结构调整选择器

### 问题 3：Project Manager 搜索
如果搜索结果为空：
- 检查 LoginID 是否正确
- 确认用户存在于系统中
